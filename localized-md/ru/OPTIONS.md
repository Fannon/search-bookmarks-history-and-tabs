# Настройки пользователя

> **Язык / Language:** [English](../../OPTIONS.md) | **Русский**
> *   **Главный Readme / Main Readme:** [README.md](../../README.md) | [README.ru.md](./README.md)

В данном документе перечислены все доступные параметры конфигурации расширения.
Вы можете настроить эти параметры на странице **Настройки** Диспетчера закладок с помощью интерактивной формы на основе схемы и синхронизированного редактора **YAML**.
Сохранять нужно только те параметры, которые вы хотите переопределить; для всех остальных будут использоваться значения по умолчанию.
Если вам нужна помощь, воспользуйтесь разделом [Генерация конфигурации с помощью ИИ](#генерация-конфигурации-с-помощью-ии).

Продвинутые пользователи могут изучить [JSON-схему](https://raw.githubusercontent.com/Fannon/search-bookmarks-history-and-tabs/main/popup/json/options.schema.json) для детального ознакомления со всеми свойствами и ограничениями.

## Параметры поиска

| Опция | Тип | По умолчанию | Описание |
| :--- | :--- | :--- | :--- |
| `searchStrategy` | string | `'precise'` | Метод поиска: `'precise'` (быстрый, только точные совпадения) или `'fuzzy'` (медленнее, поиск приблизительных совпадений с использованием [uFuzzy](https://github.com/leeoniya/uFuzzy)). |
| `searchMaxResults` | integer | `24` | Максимальное количество отображаемых результатов поиска. Меньшие значения повышают производительность. Не применяется к поиску по тегам и папкам (где всегда отображаются все совпадения). |
| `searchFuzzyness` | number | `0.6` | Допустимая нечеткость поиска (0–1). Более высокие значения находят более приблизительные совпадения, но могут возвращать менее релевантные результаты. Применяется только когда `searchStrategy` имеет значение `'fuzzy'`. |
| `openInCurrentTab` | boolean | `false` | Открывать результаты в текущей вкладке по умолчанию. Если включено, удерживайте `Shift` или `Alt`, чтобы открыть в новой вкладке (инвертирует поведение по умолчанию). |

| :--- | :--- | :--- | :--- |
| `bookmarkColor` | string | `'#3c8d8d'` | Цвет для результатов закладок (бирюзовый). |
| `tabColor` | string | `'#b89aff'` | Цвет для результатов открытых вкладок (фиолетовый). |
| `historyColor` | string | `'#9ece2f'` | Цвет для результатов истории (салатовый). |
| `searchColor` | string | `'#e1a535'` | Цвет для результатов поисковых систем (янтарный). |
| `customSearchColor` | string | `'#ce5c2f'` | Цвет для результатов пользовательских поисковых систем (оранжевый). |
| `directColor` | string | `'#7799CE'` | Цвет для результатов прямой навигации по URL (синий). |

## Источники поиска

Переключение источников данных, включаемых в результаты поиска.

| Опция | Тип | По умолчанию | Описание |
| :--- | :--- | :--- | :--- |
| `enableTabs` | boolean | `true` | Включать открытые вкладки браузера в результаты поиска. |
| `enableBookmarks` | boolean | `true` | Включать закладки в результаты поиска. |
| `enableHistory` | boolean | `true` | Включать историю посещений в результаты поиска. Примечание: API истории браузера может работать медленно; см. раздел настроек истории для оптимизации. |
| `enableSearchEngines` | boolean | `true` | Показывать ссылки на поисковые системы в качестве альтернативных результатов (например, «Искать в Google...»). |
| `enableDirectUrl` | boolean | `true` | При вводе запроса, похожего на URL (например, `example.com`), предлагать прямой переход. |
| `quickBookmarkCurrentTab` | string или `false` | `'Bookmarks bar'` | Название или ID папки, используемой по умолчанию для быстрого сохранения активной вкладки. Идентификаторы папок (ID) более надежны в локализованных версиях браузеров, чем их названия. Укажите `false`, пустую строку или только пробелы для отключения этой функции. |

## Опции поисковых систем

| Опция | Тип | По умолчанию | Описание |
| :--- | :--- | :--- | :--- |
| `searchEngineChoices` | array | См. ниже | Поисковые системы, отображаемые в качестве альтернативных результатов. Каждая запись отображается по порядку. Используйте `$s` в `urlPrefix` в качестве заполнителя для поискового запроса. |
| `customSearchEngines` | array | См. ниже | Поисковые системы, вызываемые по алиасу. Введите алиас, а затем пробел (например, `g react`), чтобы начать поиск. Поддерживает параметр `blank` для URL-адреса, если поисковый запрос не введен. |

**Опция `searchEngineChoices` по умолчанию:**
```yaml
searchEngineChoices:
  - name: Google
    urlPrefix: https://www.google.com/search?q=$s
```

**Опция `customSearchEngines` по умолчанию:**
```yaml
customSearchEngines:
  - alias: [g, google]
    name: Google
    urlPrefix: https://www.google.com/search?q=$s
    blank: https://www.google.com
  - alias: [d, dict]
    name: dict.cc
    urlPrefix: https://www.dict.cc/?s=$s
    blank: https://www.dict.cc
```

## Опции расчета оценки (Scoring)

Результаты ранжируются по итоговой оценке. Элементы с более высокими оценками отображаются первыми.

Система оценки использует **5-шаговый процесс**:
1. **Базовая оценка** — Начало с оценки на основе типа результата (закладка, открытая вкладка, история и т. д.)
2. **Качество поиска** — Умножение на коэффициент совпадения с поисковым запросом (0–1 от точного/нечеткого алгоритма)
3. **Бонусы за совпадение** — Добавление баллов за точные совпадения, вхождения подстроки, совпадения тегов/папок
4. **Сигналы использования** — Добавление баллов на основе количества посещений и новизны
5. **Ручной бонус** — Добавление пользовательского бонуса из названий закладок (например, `+20`)

Для подробного ознакомления см. [раздел «Система оценки» в README.ru.md](https://github.com/Fannon/search-bookmarks-history-and-tabs/blob/main/README.ru.md#система-оценки-релевантности-scoring).

### Базовые оценки (по типу результата)

| Опция | Тип | По умолчанию | Описание |
| :--- | :--- | :--- | :--- |
| `scoreBookmarkBase` | integer | `100` | Стартовая оценка для результатов закладок. |
| `scoreTabBase` | integer | `70` | Стартовая оценка для открытых вкладок. |
| `scoreHistoryBase` | integer | `45` | Стартовая оценка для результатов истории. |
| `scoreSearchEngineBase` | integer | `30` | Стартовая оценка для ссылок встроенных поисковых систем. |
| `scoreCustomSearchEngineBase` | integer | `400` | Стартовая оценка для результатов альтернативного поиска по алиасу (высокая для вывода наверх). |
| `scoreDirectUrlScore` | integer | `500` | Оценка для прямого перехода по URL (наивысший приоритет). |

## Фавиконы веб-сайтов

Установите **`displayFavicons: true`**, чтобы отображать фавиконы веб-сайтов рядом с результатами поиска, когда они кэшированы браузером.

### Реализация и конфиденциальность

- **Без внешних запросов**: Для обеспечения конфиденциальности эта функция разработана с полным отсутствием внешних сетевых вызовов. Значки загружаются исключительно из локального кэша вашего браузера.
- **Встроенный API Chrome/Edge**: Браузеры на базе Chromium предоставляют безопасный встроенный инструмент (`_favicon`) для извлечения кэшированных значков ваших закладок и истории.
- **Firefox и отсутствующие значки**: Firefox в настоящее время не поддерживает нативный API фавиконов для фонового доступа. Если фавикон недоступен, элемент отображается с универсальной иконкой-заглушкой.
- **Синхронизация вкладок**: Если закладка открыта в текущей вкладке, расширение копирует её текущий значок напрямую.

### Технические детали

Чтобы соответствовать принципу наименьших привилегий, разрешение **«Чтение и изменение ваших фавиконов»** обрабатывается как **необязательное разрешение**.

- Если вы включите `displayFavicons: true`, браузер предложит вам подтвердить это разрешение при сохранении настроек.
- Если вы никогда не используете фавиконы, расширение никогда не запросит это разрешение.

Технически, чтобы разрешить всплывающему окну расширения отображать эти значки, ресурс фавиконов должен быть объявлен как «доступный через веб» (web accessible). Это создает минимальный **риск конфиденциальности по побочным каналам**: вредоносный сайт, знающий ID этого расширения, теоретически может проверить, присутствует ли конкретный домен (например, `yourbank.com`) в кэше значков вашего браузера. Учитывая минимальный характер этого риска по сравнению с удобством функции, это стандартное решение в большинстве современных расширений поиска.

## Генерация конфигурации с помощью ИИ

Вы можете использовать любую большую языковую модель (например, ChatGPT, Claude или Gemini) для генерации пользовательской конфигурации под ваши задачи.

Скопируйте и вставьте следующий промпт «Эксперт по конфигурации» в ИИ:

```md
Act as an expert configuration assistant for the [Search Bookmarks, History and Tabs](https://github.com/Fannon/search-bookmarks-history-and-tabs) browser extension.

Generate a valid **YAML** user configuration for the extension.

## Rules
- Use only documented options from the official JSON Schema and documentation.
- Do not invent option names.
- Include only options that should differ from the official defaults.
- Omit any option that would keep its default value.
- Respect all schema types, allowed values, minimums, maximums, and object shapes.
- Prefer conservative settings that keep search fast unless I explicitly ask for broader/slower behavior.
- If my request is ambiguous or conflicts with the schema, ask a brief clarification question instead of guessing.
- Return the configuration as a fenced `yaml` code block.
- You may add a short explanation after the YAML block for non-obvious choices, tradeoffs, or omitted defaults.

## Sources of truth
- JSON Schema: https://raw.githubusercontent.com/Fannon/search-bookmarks-history-and-tabs/main/popup/json/options.schema.json
- Documentation: https://raw.githubusercontent.com/Fannon/search-bookmarks-history-and-tabs/main/OPTIONS.md

## What I want

E.g. I want to prioritize open tabs over everything else.
I want to use fuzzy search and history for only the last 7 days.
```
