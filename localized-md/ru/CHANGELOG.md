## [В разработке (Unreleased)]

- **УЛУЧШЕНО**: Экспорт закладок в HTML теперь включает идентификатор (`ID`) браузерной закладки для каждого элемента `<A>` и `<H3>`. Этот атрибут безопасно игнорируется при повторном импорте и позволяет внешним инструментам (например, файлам предложений очистки ИИ) ссылаться на закладки по их реальным ID.

## [v2.4.0] - 2026-06-27

- **НОВОЕ**: Добавлены быстрые действия для сохранения закладок и их редактирования прямо из результатов поиска.
  - По умолчанию первый результат теперь позволяет сохранить текущую вкладку через редактор закладок.
  - Нажмите `F2` во всплывающем окне поиска, чтобы изменить выбранную закладку или создать новую из выбранного URL.
  - Папку для быстрого сохранения можно задать через `quickBookmarkCurrentTab` (укажите пустую строку для отключения).
- **НОВОЕ**: Настройки пользователя перенесены в полностраничный Диспетчер закладок на отдельную вкладку «Настройки».
  - Добавлена интерактивная форма настроек на основе схемы, описания параметров, валидация «на лету» и редактор YAML.
  - Переход к настройкам из контекстного меню теперь открывает `popup/bookmarkManager.html#options` в новой вкладке.
- **НОВОЕ**: Добавлена опция `openInCurrentTab` для открытия выбранных результатов в текущей вкладке по умолчанию (удержание `Shift` или `Alt` инвертирует поведение).
- **ИСПРАВЛЕНО**: Состояние результатов поиска теперь обновляется более надежно после закрытия вкладок, изменения закладок и сбоев нечеткого поиска.
- **ИСПРАВЛЕНО**: Открытие всплывающего окна по маршруту `#search/...` теперь автоматически запускает этот поиск на старте.
- **ИСПРАВЛЕНО**: При некорректных сохраненных настройках теперь используются значения по умолчанию и показывается скрываемый оверлей ошибок.
- **УДАЛЕНО**: Удален параметр `detectDuplicateBookmarks` и индикатор дубликатов во всплывающем окне. Очистка дубликатов по-прежнему доступна в Диспетчере закладок.
- **УДАЛЕНО**: Удален параметр `displayIcons` и плейсхолдеры типов результатов в поиске.
- **УЛУЧШЕНО**: Повышена производительность запуска всплывающего окна и поиска, расширены бенчмарки для больших объемов данных.

## [v2.3.0] - 2026-05-09

- **НОВОЕ**: Добавлен бета-версия Диспетчера закладок в виде полностраничного режима для просмотра статистики закладок, навигации по папкам, изменения метаданных, перемещения, управления тегами и очистки дубликатов URL.
  - Поддерживается экспорт закладок в HTML и сохранение последних 50 действий для отмены (undo) в памяти.
  - Встроен Диспетчер тегов для фильтрации, переименования и удаления тегов во всех совпадающих закладках.
  - Интегрированы опциональные подсказки тегов от локальной нейросети браузера (через API `LanguageModel`).
- **НОВОЕ**: Добавлена вкладка «AI-очистка» в Диспетчере закладок.
  - Возможность генерации быстрых (Lite) или полных (Advanced) промптов для внешних ИИ-сервисов.
  - Локальный ИИ браузера может составить JSON-предложение по группировке, переименованию тегов и удалению дубликатов.
- **ИСПРАВЛЕНО**: Повторные попытки подсказки тегов локальным ИИ для большого количества выбранных закладок теперь корректно ограничивают контекст первыми 20 закладками.

## [v2.2.0] - 2026-05-05

- **НОВОЕ**: Добавлена система оценки избранного на основе звездочек. В редакторе закладок кнопка FAVORITE переключает четыре состояния: без избранного, желтая звезда (`+25`), оранжевая звезда (`+50`) и красная звезда (`+75`). Балл сохраняется в названии закладки в формате `+N`.
- **ИСПРАВЛЕНО**: Игнорирование горячих клавиш навигации (Enter, стрелки) во время ввода текста с использованием IME (предотвращает случайный выбор результатов у пользователей CJK-ввода).
- **ИСПРАВЛЕНО**: Сохранение оригинальных URL-адресов закладок, истории и вкладок для отображения и открытия, при сохранении нормализованных URL для поиска и дедупликации.

## [v2.1.1] - 2026-03-24

- **УЛУЧШЕНО**: Производительность: Уменьшена избыточная работа при точном инкрементальном поиске путем пропуска неизменившихся начальных условий.
- **УЛУЧШЕНО**: Производительность: Заменена полная сортировка результатов выбором top-k элементов в пределах лимита `searchMaxResults`.
- **УЛУЧШЕНО**: Производительность: Отключение генерации разметки подсветки, если отключен параметр `displaySearchMatchHighlight`.
