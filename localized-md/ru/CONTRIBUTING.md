## Локальная разработка

### Установка и сборка

- Требования: [Node.js](https://nodejs.org/en/) v22 или новее и bash-совместимый терминал.
- Установите зависимости с помощью `npm install`.
- Соберите бандлы командой `npm run build`. Это сгенерирует оптимизированный код в папке `dist/`.
- Локально собранное расширение можно установить/загрузить в браузер как из корня проекта (для отладки исходного кода), так и из директории `/dist/chrome` (для проверки оптимизированной продакшн-сборки).
- Для автоматической сборки при редактировании файлов запустите `npm run watch`.
- Вы можете запустить локальный сервер для отладки всплывающего окна напрямую через `npm run start` или проверить сборку дистрибутива с помощью `npm run start:dist`.
- Используйте `npm run clean` для сброса результатов сборки при возникновении неконсистентностей.

### Структура проекта

- `popup/` содержит HTML, CSS, входные скрипты и библиотеки всплывающего окна.
  - `popup/js/helper/` содержит общие утилиты.
  - `popup/js/model/` описывает модели данных и настройки по умолчанию (см. `options.js`).
  - `popup/js/search/` координирует логику парсинга и оркестрации поиска.
  - `popup/js/view/` рендерит пользовательский интерфейс и связывает события с обновлениями DOM.
  - `popup/js/**/__tests__/` группирует модульные тесты Jest рядом с соответствующими модулями.
- `playwright/tests/` содержит сквозные сценарии тестирования (E2E).
- `images/` содержит статические ресурсы расширения.
- `dist/` содержит оптимизированную сборку для публикации (`dist/chrome/`). Firefox теперь использует тот же манифест Chrome.

### Установка в режиме разработчика

- **Chrome / Edge**
  - Откройте страницу `chrome://extensions/` (Chrome) или `edge://extensions/` (Edge).
  - Включите «Режим разработчика» в правом верхнем углу.
  - Нажмите кнопку «Загрузить распакованное расширение» и выберите папку `dist/chrome/`.
- **Firefox**
  - После запуска сборки через `npm run build`, откройте страницу `about:debugging`.
  - Перейдите во вкладку «Этот Firefox» (This Firefox) → «Загрузить временный флагман…» (Load Temporary Add-on…) и выберите файл `manifest.json` внутри папки `dist/chrome/` (сборка Chrome отлично работает для временных установок в Firefox).
  - Временные расширения автоматически удаляются при перезапуске браузера.
- **Применение изменений**
  - Перезапускайте `npm run build` перед обновлением расширения в браузере или держите запущенным `npm run watch`, чтобы файлы в `dist/` автоматически обновлялись при сохранении исходного кода.

### Рабочий процесс разработчика

- `npm run build` — Полный конвейер сборки (очистка → бандлинг → обновление манифестов → упаковка dist → отчет о размере).
- `npm run watch` — Пересборка бандлов при сохранении изменений файлов.
- `npm run start` / `npm run start:dist` — Запуск локального сервера для отладки исходников или дистрибутива.
- `npm run test` или `npm run test:unit` — Запуск юнит-тестов Jest; `npm run test:unit:coverage` для отчета о покрытии тестами.
- `npm run test:e2e` — Запуск сквозных тестов Playwright.
- `npm run lint` — Запуск проверки кода Biome (форматирование и линтинг).
- `npm run analyze` — Запуск скрипта анализа кода (`bin/analyze-code.js`).
- `npm run size` — Отчет о размерах файлов сборки.

Смотрите также: [AGENTS.md](./AGENTS.md) для ознакомления с архитектурой, процессами верификации и требованиями к производительности.

## Как внести свой вклад

1. **Сделайте форк репозитория** (Fork) и создайте ветку от `main`.
2. **Установите зависимости:**

```bash
npm install
```

3. **Соберите и запустите проект локально:**

- Соберите проект один раз: `npm run build`. Это очистит рабочую область, обновит манифесты, сгруппирует скрипты и скопирует результаты в `dist/`.
- Используйте `npm run watch` при написании кода. Он автоматически пересобирает бандлы, отражая изменения в директории `dist/`.
- Откройте локальный просмотр UI всплывающего окна: `npm run start` (запускает сервер на http://localhost:8080).
- Для инструкций по загрузке в браузер см. раздел [Установка в режиме разработчика](#установка-в-режиме-разработчика).

4. **Тестирование:**

- Проверка стиля кода: `npm run lint` (Biome проверит весь репозиторий).
- Запуск тестов Jest: `npm run test:unit`. Можно запустить тесты для конкретного файла: `npm run test:unit <путь_к_тесту>.test.js`.
- Запуск E2E-тестов Playwright: `npm run test:e2e` (также доступны тесты для отдельных браузеров, например `test:e2e:chromium`).

5. **Стиль кода и паттерны:**

- Следуйте модульной структуре и правилам именования из [AGENTS.md](./AGENTS.md).
- Размещайте логику поиска в `popup/js/search/`, представления (рендеринг) в `popup/js/view/`, модели данных в `popup/js/model/`, а утилиты — в `popup/js/helper/`.
- Не добавляйте новые зависимости без веских причин и предварительного согласования; пишите компактный и совместимый с расширениями код.

6. **Добавление новых возможностей:**

- При внедрении новых режимов поиска обновите парсинг запросов в `popup/js/search/queryParser.js` и соответствующие файлы логики поиска.
- Отражайте изменения в файлах `README.md` и `OPTIONS.md`, если они добавляют новые настройки пользователя.

7. **Пулл-реквесты (Pull Requests):**

- Убедитесь, что ваш код покрыт тестами, отформатирован и документирован.
- Выполните `npm run lint`, `npm run test:unit` и `npm run test:e2e` перед открытием PR.
- При изменении алгоритмов поиска или рендеринга запустите `npm run test:perf` для исключения деградации производительности.
- Указывайте связанные задачи (Issues) в описании пулл-реквеста.
- Старайтесь делать PR сфокусированными на одной конкретной задаче.
